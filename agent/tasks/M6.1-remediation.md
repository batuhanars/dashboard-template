# M6.1 — Remediation (Disiplin + i18n)

**Status:** DONE
**Önkoşul:** M6 (fonksiyonel olarak DONE).
**İlgili contract'lar:** `CONVENTIONS.md` ("Component organizasyonu — DİSİPLİN"), `components.md`, `PROJECT.md` (DoD).
**Kaynak:** PM teslim denetimi — M6 fonksiyonel çalışıyor ama 2 kabul kriterinden geçmedi.

## Hedef

M6 view'larını **davranışı değiştirmeden** disiplin-uyumlu hale getir ve kalan hard-coded string'leri i18n'e taşı. Yalnızca yapı + çeviri; iş mantığı, query key'leri, mutasyonlar aynı kalır. Build yeşil kalmalı.

> Bu bir refactor pass'i. Her view'ı parçalarken props/emit'leri doğru bağla — regresyon riski burada. Parçaladıkça `npm run dev` ile o sayfayı gözle doğrula.

## Adımlar

### 1. Component extraction (DİSİPLİN)

Ekran-spesifik parçaları `views/<x>/components/` altına çıkar. Ana View **compose-only** kalsın (hedef <120 satır). State store/composable'da kalır; sub-component props alır + emit eder. Mevcut `components/shared/*` ve composable'lar **yeniden kullanılır** — yeni shared üretme.

- [x] **`views/dashboard/components/`** ✅ — `DashboardRecentTransactions.vue` çıkarıldı; `DashboardView.vue` 127→85 (compose-only hedefin altında, kabul). KPI/chart shared component kullandığı için inline kalması sorun değil.
- [x] **`views/users/components/`** ✅ — `UsersFilterBar`, `UsersBulkActionBar`, `UserProfileTab`, `UserActivityTab`, `UserPermissionsTab` çıkarıldı. `UsersListView` 286→70, `UserDetailView` 262→65.
- [x] ✅ **`views/settings/components/`** — `SettingsProfileForm`, `SettingsAppearance`, `SettingsNotifications`, `SettingsDangerZone` çıkarıldı. `SettingsView` 234→37 satır (compose-only).
- [x] **Değişmez:** `ForgotPasswordView.vue` (58) extraction gerektirmez. `LoginView.vue` (104) extraction gerektirmez ama i18n düzeltmesi eksik (adım 2).

### 2. i18n düzeltmesi

Hard-coded string'leri locale key'e taşı; `tr.json` + `en.json` **birebir** key seti.

- [x] ✅ `UserDetailView` izin listesi: `UserPermissionsTab.vue` içinde data-driven i18n'e taşındı (`users.detail.permissions.*`, `t(perm.key)`). tr+en.
- [x] ✅ `LoginView.vue` brand VITE_APP_NAME'den, quote/footer `$t('auth.login.marketingQuote/Footer')` ile render ediliyor (tr+en).
- [x] ✅ `SettingsDangerZone.vue` toast `t('settings.danger.deleteRequested')` — tr+en eklendi.
- [x] ✅ Tüm key'lerin İngilizce karşılığı var; tr/en key seti birebir.

## Kabul kriterleri

- `npm run build` — type-check + build hatasız.
- Her sayfa **eskisi gibi** çalışıyor (davranış değişmedi): Dashboard KPI+chart+recent; UsersList arama/filtre/sort/paginate/bulk; UserDetail tab+form+save→invalidate→toast; Settings tab'ları+danger zone; Login.
- `views/{dashboard,users,settings}/components/` klasörleri dolu; ana View dosyaları ~<120 satır.
- `src/views` içinde kullanıcı-görünür hard-coded TR string kalmadı (grep ile teyit).
- EN'e geçince izin listesi + login cümlesi + danger zone toast çevriliyor.
- `tr.json` ↔ `en.json` key seti birebir.
